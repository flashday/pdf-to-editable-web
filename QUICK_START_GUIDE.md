# 快速启动指南 - PDF 转可编辑网页系统

## 🎉 系统已启动成功！

### 访问地址

- **前端界面**: http://localhost:3000
- **后端API**: http://localhost:5000/api
- **健康检查**: http://localhost:5000/health

### 如何使用

1. **打开浏览器**
   - 访问 http://localhost:3000
   - 你会看到一个文件上传界面

2. **上传文档**
   - 点击上传区域或拖拽文件
   - 支持的格式：PDF、JPG、PNG
   - 文件大小限制：10MB
   - 注意：多页PDF只会处理第一页

3. **查看转换结果**
   - 系统会自动处理文档
   - 显示实时处理状态
   - 完成后会在Editor.js编辑器中显示结果

4. **编辑内容**
   - 点击任何文本块即可编辑
   - 支持标题、段落、表格等多种内容类型
   - 表格可以编辑单元格内容
   - 所有更改都是实时的

### 停止系统

要停止系统，你需要停止两个进程：

**方法1：使用进程管理**
```bash
# 查看运行的进程
ps aux | grep python
ps aux | grep node

# 停止进程
kill <进程ID>
```

**方法2：在终端中**
- 如果在终端中运行，按 `Ctrl+C` 停止

### 测试API

你可以使用curl测试API：

```bash
# 健康检查
curl http://localhost:5000/health

# 上传文件（需要实际的PDF文件）
curl -X POST http://localhost:5000/api/convert \
  -F "file=@your_document.pdf"

# 检查状态
curl http://localhost:5000/api/convert/<job_id>/status

# 获取结果
curl http://localhost:5000/api/convert/<job_id>/result
```

### 故障排除

**如果前端无法连接后端：**
1. 检查后端是否在运行：`curl http://localhost:5000/health`
2. 检查CORS配置是否正确
3. 查看浏览器控制台的错误信息

**如果OCR处理失败：**
1. 检查PaddleOCR是否正确安装
2. 查看后端日志中的错误信息
3. 确认文件格式正确且未损坏

**如果内存不足：**
1. 系统会自动限制内存使用
2. 尝试处理较小的文件
3. 检查系统资源使用情况
