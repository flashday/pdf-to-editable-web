# 演示说明

## ✅ 系统已成功启动！

### 当前运行状态

🟢 **后端服务器**: http://localhost:5000 (运行中)  
🟢 **前端界面**: http://localhost:3000 (运行中)  
🟢 **API端点**: http://localhost:5000/api (可用)

### 立即体验

1. **打开浏览器访问**: http://localhost:3000

2. **你会看到**:
   - 一个简洁的文件上传界面
   - 支持拖拽上传
   - 实时状态显示

3. **上传测试文档**:
   - 准备一个PDF、JPG或PNG文件
   - 拖拽到上传区域或点击选择
   - 系统会自动开始处理

4. **观察处理过程**:
   - 上传进度条
   - OCR处理状态
   - 实时进度更新

5. **查看和编辑结果**:
   - 转换完成后自动显示在编辑器中
   - 点击任何文本即可编辑
   - 表格支持单元格编辑
   - 可以添加、删除、修改内容

### 系统特性演示

#### 1. 文件验证
- 尝试上传不支持的格式 → 会显示友好的错误提示
- 尝试上传超过10MB的文件 → 会提示文件过大

#### 2. OCR处理
- 上传扫描的PDF → 系统会识别文本
- 上传包含表格的文档 → 表格结构会被保留
- 查看置信度分数 → 了解识别准确度

#### 3. 实时编辑
- 点击标题 → 可以修改标题文本
- 点击段落 → 可以编辑段落内容
- 点击表格单元格 → 可以修改单元格内容

#### 4. 错误处理
- 网络中断时 → 自动重试
- 处理失败时 → 显示详细错误信息
- 低置信度时 → 显示警告提示

### 技术亮点

✨ **智能OCR**: 使用PaddleOCR PP-Structure进行高精度识别  
✨ **结构保留**: 保持原文档的布局和层次结构  
✨ **实时反馈**: WebSocket/轮询提供实时处理状态  
✨ **容错机制**: 自动重试和错误恢复  
✨ **性能优化**: 内存限制和资源自动清理  

### 查看运行日志

**后端日志** (在启动后端的终端中):
- 显示所有API请求
- OCR处理进度
- 错误和警告信息
- 性能指标

**前端日志** (浏览器开发者工具):

### 停止系统

当你完成演示后，可以停止系统：

1. 在Kiro中找到正在运行的进程
2. 或者在终端中按 `Ctrl+C`
3. 或者使用命令：
   ```bash
   # 查找进程
   lsof -i :5000  # 后端
   lsof -i :3000  # 前端
   
   # 停止进程
   kill <PID>
   ```

### 测试建议

推荐测试文档类型：
- ✅ 简单的文本PDF
- ✅ 包含标题和段落的文档
- ✅ 包含表格的文档
- ✅ 扫描的图片文档
- ✅ 多页PDF（只会处理第一页）

### 已知限制

- 📄 多页PDF只处理第一页
- 📏 文件大小限制10MB
- 🖼️ 图片内容会被识别为文本
- 🎨 不保留原始格式样式（颜色、字体等）

### 问题反馈

如果遇到问题：
1. 检查浏览器控制台的错误信息
2. 查看后端终端的日志
3. 确认文件格式和大小符合要求
4. 尝试使用不同的测试文档
